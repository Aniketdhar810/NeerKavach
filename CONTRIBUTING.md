# Contributing Guide
Thank you for your interest in contributing.

## Code of Conduct
Please read the CODE_OF_CONDUCT.md before contributing.

## Reporting Bugs
Create a GitHub issue and include:
- Steps to reproduce
- Expected behavior
- Screenshots if applicable

## Suggesting Features
Open an issue labeled "enhancement".

## Development Setup

1. Clone the repository
2. Install dependencies
3. Run frontend and backend servers

## Coding Standards
- Follow ESLint rules
- Use consistent formatting
- Write clean and modular code

## Branch Naming

feature/new-feature  
bugfix/fix-login  
docs/update-readme

## Commit Messages

Use Conventional Commits:

feat: add prediction API  
fix: correct dashboard bug

## Pull Request Process

1. Fork repository
2. Create feature branch
3. Commit changes
4. Open Pull Request
